# Quick Start

Connect to a Salesforce Trailhead org and authenticate. Then make a request to the CRM Analytics REST API in Postman, and look at the response.

-   **[Prerequisites](atlas.en-us.bi_dev_guide_rest.meta/bi_dev_guide_rest/prerequisites.htm)**
    Complete these prerequisites before you begin the quick start.
-   **[Configure Your Trailhead Org](atlas.en-us.bi_dev_guide_rest.meta/bi_dev_guide_rest/bi_configure_your_trailhead_org.htm)**
    To configure your Trailhead org, set up Cross-Origin Resource Sharing (CORS) and a connected app.
-   **[Authenticate to Your Trailhead Playground Org](atlas.en-us.bi_dev_guide_rest.meta/bi_dev_guide_rest/authenticate_to_your_trailhead_playground_org.htm)**
    After configuring the connected app, you must authorize your user with the connected app.
-   **[Set Up Your Postman Collection](atlas.en-us.bi_dev_guide_rest.meta/bi_dev_guide_rest/set_up_postman_collection.htm)**
    Set up Postman to work with the Salesforce CRM Analytics Connect API collection.
-   **[Use Postman to Send a CRM Analytics Request](atlas.en-us.bi_dev_guide_rest.meta/bi_dev_guide_rest/bi_send_postman_request.htm)**
    With Postman, you can explore and test your CRM Analytics API calls across multiple organizations with full control on headers, parameters, and content type.
