# Debug Lightning Web Components in the Field Service Mobile App

The best way to develop and debug your Lightning web components is the same way you develop and debug *anything* built with HTML, CSS, and JavaScript: with the debugging tools built into your web browser.

-   For debugging on Android, use [Chrome DevTools](https://developer.chrome.com/docs/devtools/ "HTML (New Window)")
-   For debugging on iOS, use [Safari Web Inspector](https://developer.apple.com/safari/tools/ "HTML (New Window)")

Next, this section walks you through connecting your desktop browser’s developer tools to the WebView within the Field Service mobile app. From there, debugging an LWC running on a mobile device is like any other web app debugging session.

-   **[Install Local Development Server Plugin](atlas.en-us.field_service_dev.meta/field_service_dev/sfs_gs_lwc_debugging_setup.htm)**
    Whether you’re debugging your component for iOS or Android, you must first install the LWC Development Server for mobile.
-   **[Debug in iOS](atlas.en-us.field_service_dev.meta/field_service_dev/sfs_gs_lwc_debugging_ios.htm)**
    Connect Safari Web Inspector on your desktop to the WebView in the Field Service mobile app where your LWC is running.
-   **[Debug in Android](atlas.en-us.field_service_dev.meta/field_service_dev/sfs_gs_lwc_debugging_android.htm)**
    Connect Chrome DevTools on your desktop to the WebView in the Field Service mobile app where your LWC is running.
