# REST Reference

You can access Health Cloud Business APIs using REST endpoints. These REST APIs follow similar conventions as Connect REST APIs.

To understand the architecture, authentication, rate limits, and how the requests and responses work, see [Connect REST API Developer Guide](https://developer.salesforce.com/docs/atlas.en-us.260.0.chatterapi.meta/chatterapi/intro_what_is_chatter_connect.htm "HTML (New Window)")

-   **[Resources](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_resources.htm)**
    Use these resources of Health Cloud Business APIs to enhance your Salesforce implementation or integrate with other applications.
-   **[Request Bodies](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_requests.htm)**
    This section lists the request bodies for Health Cloud Business APIs. The query parameters for an endpoint are listed along with each resource.
-   **[Response Bodies](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_responses.htm)**
    A resource can return a response body in either JSON or XML format.
