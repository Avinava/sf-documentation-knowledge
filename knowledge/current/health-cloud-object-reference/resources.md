# Resources

Use these resources of Health Cloud Business APIs to enhance your Salesforce implementation or integrate with other applications.

-   **[Appointment Management](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_appointment_management_intro.htm)**
    Book, cancel or update an appointment for a patient.
-   **[Assessment Previous Responses](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_assessment_previous_responses.htm)**
    Retrieves historical assessment response data for healthcare applications, supporting both internal Discovery Framework and external Milliman Care Guidelines (MCG) assessments.
-   **[Book Slot Chain (POST)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_book_slot_chain.htm)**
    Create service appointments for a care program enrollee in advanced therapy management based on the provided appointment slots.
-   **[Care Services Review and Response](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_care_services_review_intro.htm)**
    These RESTful APIs support data elements required for ANSI X12 278 format Health Care Services Review and Response. Also referred to as ANSI 278 Authorization APIs, they enable payer organizations to electronically receive prior authorization requests and referral requests from providers. Other users of these APIs can be healthcare providers, payers, plan sponsors, or other entities involved in health care services review, such as clearinghouses.
-   **[Contact Encounter (POST)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_erm_resource_contact_encounter_business_api.htm)**
    Load details of contact encounters into the system.
-   **[Care Program Enrollments (POST)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_careprogram_enrollment_intro.htm)**
    Use this API to enroll multiple patients in a care program.
-   **[Digital Verification (POST)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_digital_verification.htm)**
    Perform verification for the digital verification records within advanced therapy management.
-   **[Guest Referral](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_guest_referral.htm)**
    Create a patient referral in a Salesforce org from an OOTB guest web form.
-   **[Identity Verification](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/identity_verification_api_intro.htm)**
    Use these resources to verify the caller's identity. For example, the call center agent can ask the person to confirm their social security number, birth date, insurance details, or driver’s license number.
-   **[Benefit Verification](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/benefit_verification_api_intro.htm)**
    Use these resources to verify the health benefits covered by the patient's insurer. The call center agent can view the coverages in the patient's record page.
-   **[Intelligent Appointment Management Operations](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/healthcare_appointment_booking_operations.htm)**
    The available operations for Intelligent Appointment Management include finding available appointment time slots, determining the status of an appointment slot, creating an appointment, getting a scheduled appointment, and canceling an appointment.
-   **[Intelligent Document Management](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/intelligent_doc_management_apis.htm)**
    Use the Intelligent Document Management resources to create or update records in Salesforce objects based on the content document records or received document records.
-   **[Medication Statements](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/med_statement_api_intro.htm)**
    Use these resources to create a medication statement or update an existing medication statement for a patient.
-   **[Omniscript](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_store_discovery_framework_structure.htm)**
    Get the discovery framework structure stored as OmniProcess.
-   **[Patients (GET, POST, PUT)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_create_get_update_patients.htm)**
    Retrieve the FHIR patient resource record details. Create or update a patient record rather than making multiple separate calls to the objects in Salesforce.
-   **[Providers](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_providers_api_intro.htm)**
    Use these resources to create practitioner or facility records, or update existing fields in provider records and insert new fields.
-   **[Referral Management](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_referal_management.htm)**
    Create a patient referral in a Salesforce org from an external data source.
-   **[Unified Health Scoring](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/hc_unified_health_scoring_apis.htm)**
    Unified Health Scoring (UHS) assists payers and providers by giving them a holistic view into their patients’ health profiles.
-   **[Visits (POST)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_create_visit.htm)**
    Create visits for sales representatives. For example, for a medical device visit, a sales representative visits the surgeon or a hospital location.
-   **[Work Type Lead Time (POST)](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_work_type_lead.htm)**
    Get a list of work types and their associated lead times that are optimized across regions for a specific advanced therapy.
-   **[Quotas and Allocations](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/connect_resources_quotas_and_allocations.htm)**
    Find available slots and manage quotas for therapies (Work Procedures) at various stages (Work Types) across multiple treatment centers.
