# Documentation Templates and Rules

Da Vinci Documentation Templates and Rules (DTR) is a framework under the HL7 FHIR standards that facilitates the exchange of documentation requirements in the healthcare industry. DTR enables the use of standardized templates and rules in the exchange of critical healthcare documentation, ensuring compliance and reducing administrative burden.

| Available in: Enterprise and Unlimited editions of Health Cloud |
| --- |


Documentation Templates and Rules uses objects in the Coverage Requirement Discovery (CRD) data model. To learn more about the Coverage Requirement Discovery objects, see [Coverage Requirement Discovery](https://developer.salesforce.com/docs/atlas.en-us.260.0.health_cloud_object_reference.meta/health_cloud_object_reference/hc_coverage_requirement_discovery_data_model.htm).

Documentation Templates and Rules objects are available to users with the Hls Clinical Decision Support permission set.

![Documentation Templates and Rules Diagram](/docs/resources/img/en-us/260.0?doc_id=dev_guides%2Fhealthcare_api%2Fimages%2Fsforce_hc_dtr_erd.png&folder=health_cloud_object_reference)

-   **[ServiceInfoRequestOperation](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/sforce_api_objects_serviceinforequestoperation.htm)**
    Represents the system operation performed for a service information request such as a session launch or a questionnaire package operation. This object is available in API version 63.0 and later.
-   **[ServiceInfoRqstOpOutcome](atlas.en-us.health_cloud_object_reference.meta/health_cloud_object_reference/sforce_api_objects_serviceinforqstopoutcome.htm)**
    Represents the outcome of a service information request operation. This object is available in API version 63.0 and later.
