# AJAX Toolkit Developer Guide

Embed API calls in Visualforce pages, buttons, and links with AJAX Toolkit — a JavaScript wrapper around SOAP API.

The AJAX Toolkit supports Microsoft® Internet Explorer® versions 9, 10, and 11 with the latest Microsoft hot fixes applied, and Mozilla® Firefox®, most recent stable version. The AJAX Toolkit is based on the partner WSDL. Because there’s no type checking in JavaScript, the type information available in the enterprise WSDL isn’t needed.

![Note](/docs/resources/img/en-us/260.0?doc_id=images%2Ficon_note.png&folder=ajax)

#### Note

Before you use the AJAX Toolkit, familiarize yourself with JavaScript and with the information about SOAP API in the [SOAP API Developer Guide](https://developer.salesforce.com/docs/atlas.en-us.260.0.api.meta/api/ "HTML (New Window)").

-   **[When to Use the AJAX Toolkit](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_when_to_use.htm)**
    For best performance, use the AJAX Toolkit when working with small amounts of data.
-   **[Working with the AJAX Toolkit](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_working.htm)**

-   **[API Calls and the AJAX Toolkit](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_calls.htm)**
    This toolkit supports all SOAP API calls, as well as runTests() from Apex.
-   **[SOAP (AJAX) API End-of-Life Policy](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_eol.htm)**
    See which SOAP API versions are supported, unsupported, or unavailable.
-   **[SOAP Header Options with the AJAX Toolkit](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_headers.htm)**
    All header options in the SOAP API are supported in the toolkit, but they are specified differently than in the API.
-   **[Error Handling with the AJAX Toolkit](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_errors.htm)**
    The AJAX Toolkit provides the ability to handle errors for synchronous and asynchronous calls.
-   **[Advanced Topics](atlas.en-us.ajax.meta/ajax/sforce_api_ajax_advanced.htm)**
