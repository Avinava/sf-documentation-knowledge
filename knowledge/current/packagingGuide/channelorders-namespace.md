# CHANNEL\_ORDERS Namespace

The CHANNEL\_ORDERS namespace provides classes for submitting orders to Salesforce Partner Operations. After you send an order, you can use other classes in the namespace to edit, recall, or clone the order.

To use CHANNEL\_ORDERS namespace classes, you must have Channel Order App v2 or later installed in your Salesforce org. For information on how to invoke methods defined in managed packages, refer to the [Apex Developer Guide](https://developer.salesforce.com/docs/atlas.en-us.260.0.apexcode.meta/apexcode/ "HTML (New Window)").

The following classes are in the CHANNEL\_ORDERS namespace.

-   **[COA\_ServiceOrderSubmit Class](atlas.en-us.packagingGuide.meta/packagingGuide/apex_class_COA_ServiceOrderSubmit.htm)**
    Submit orders to Salesforce Partner Operations for processing and activation.
-   **[COA\_ServiceOrderEdit Class](atlas.en-us.packagingGuide.meta/packagingGuide/apex_class_COA_ServiceOrderEdit.htm)**
    Edit orders that you’ve submitted to Salesforce Partner Operations.
-   **[COA\_ServiceOrderRecall Class](atlas.en-us.packagingGuide.meta/packagingGuide/apex_class_COA_ServiceOrderRecall.htm)**
    Recall orders that you’ve submitted to Salesforce Partner Operations.
-   **[COA\_ServiceOrderClone Class](atlas.en-us.packagingGuide.meta/packagingGuide/apex_class_COA_ServiceOrderClone.htm)**
    Clone an existing order in the org where the Channel Order App (COA) is installed.
