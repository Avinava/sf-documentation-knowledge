# Troubleshoot an Expired Security Review

The most common reasons that an AppExchange security review expires are a missed re-review, overdue review fees, and unpaid revenue sharing. Learn how to troubleshoot the cause of an expired review.


| User Permissions Needed |
| --- |
| To access the AppExchange Partner Console: | Manage Listings |

1.  Log in to the [Salesforce Partner Community](https://partners.salesforce.com/ "HTML (New Window)").
2.  Click **Publishing** | **Technologies** | **Solutions**.
3.  Locate the solution version (1)
4.  Confirm that you see Expired in the Security Review column (2).

    ![A screenshot of the Technologies page showing a sample solution with multiple package versions, and the security review status of each version](/docs/resources/img/en-us/260.0?doc_id=appexchange%2Fsecurity_review%2Fimages%2Fsecurity_review_status_expired.png&folder=packagingGuide)

5.  Click **View Details**.
6.  Review the What happens next? (1) and Opportunities (2) sections of the page for troubleshooting guidance.

    ![The security review Overview page for an expired review describes what happens next, and lists opportunities for how to remedy the issue.](/docs/resources/img/en-us/260.0?doc_id=appexchange%2Fsecurity_review%2Fimages%2Fsecurity_review_overview_expired.png&folder=packagingGuide)
