# Session Security

After logging in, a user establishes a session with the platform. Use session security to limit exposure to your network when a user leaves the computer unattended while still logged in. Session security also limits the risk of internal attacks such as when one employee tries to use another employee’s session. Choose from several session settings to control session behavior.

You can control when an inactive user session expires, set trusted IP address ranges, and restrict access to resources based on session security. To learn more about these session security features, see these topics.

-   [Modify Session Security Settings](https://help.salesforce.com/articleView?id=xcloud.admin_sessions.htm&type=5&language=en_US "HTML (New Window)")
-   [Set Trusted IP Ranges for Your Organization](https://help.salesforce.com/articleView?id=xcloud.security_networkaccess.htm&type=5&language=en_US "HTML (New Window)")
-   [Require High-Assurance Session Security for Sensitive Operations](https://help.salesforce.com/articleView?id=xcloud.security_auth_require_ha_session.htm&type=5&language=en_US "HTML (New Window)")

You can also monitor active sessions and session details through User Sessions. For more information, check out these topics.

-   [User Sessions](https://help.salesforce.com/articleView?id=xcloud.security_user_session_info.htm&type=5&language=en_US "HTML (New Window)")
-   [User Session Types](https://help.salesforce.com/articleView?id=xcloud.security_session_types.htm&type=5&language=en_US "HTML (New Window)")
