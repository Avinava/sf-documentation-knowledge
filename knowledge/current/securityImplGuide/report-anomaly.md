# Report Anomaly

> An anomaly is any user activity that is sufficiently different from the
    historical activity of the same user. We use the metadata in Salesforce Core application logs
    about report generation and surrounding activities to build a baseline model of the historical
    activity. We then compare any new report generation activity against this baseline to determine
    if the new activity is sufficiently different to be called an anomaly. We don't look at the
    actual data that a user interacts with— we look at how the user interacts with the
    data.

# Report Anomaly

An *anomaly* is any user activity that is sufficiently different from the historical activity of the same user. We use the metadata in Salesforce Core application logs about report generation and surrounding activities to build a baseline model of the historical activity. We then compare any new report generation activity against this baseline to determine if the new activity is sufficiently different to be called an anomaly. We don't look at the actual data that a user interacts with— we look at *how* the user interacts with the data.
