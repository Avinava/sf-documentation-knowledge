# Automotive Cloud Fields on Standard Objects

Automotive Cloud adds standard fields to some standard Salesforce objects to represent information specific to vehicles and business partners. These fields are available only in orgs where Automotive Cloud is enabled.

| Available in: Enterprise, Unlimited, and Developer Editions. |
| --- |


-   **[Automotive Cloud Fields on ApplicationForm](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_applicationform.htm)**
    Standard fields extend the ApplicationForm object for use in Automotive Cloud to represent high-level information of an application that's submitted for an automotive financial product.
-   **[Automotive Cloud Fields on ApplicationFormProduct](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_applicationformproduct.htm)**
    Standard fields extend the ApplicationFormProduct object for use in Automotive Cloud to represent the relationship between an application form product and a deviation.
-   **[Automotive Cloud Fields on ApplicationFormProductProposal](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_applicationformproductproposal.htm)**
    Standard fields extend the ApplicationFormProductProposal object for use in Automotive Cloud to represent the terms attached to a scheme to sanction an automotive loan or lease.
-   **[Automotive Cloud Fields on ApplicationFormSellerItem](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_applicationformselleritem.htm)**
    Standard fields extend the ApplicationFormSellerItem object for use in Automotive Cloud to represent a junction between an application form and an automotive seller product.
-   **[Automotive Cloud Fields on Asset](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_asset.htm)**
    Standard fields extend the Asset object for use in Automotive Cloud to represent information about each physical item that’s manufactured, installed, or sold, such as a vehicle, a part like a car engine, or an accessory like car seat covers. Use the VehicleId field on asset to represent an asset as a vehicle.
-   **[Automotive Cloud Fields on BusinessProfile](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_businessprofile.htm)**
    Standard fields extend the BusinessProfile object for use in Automotive Cloud to represent information about partner account details.
-   **[Automotive Cloud Fields on InternalOrganizationUnit](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_internalorganizationunit.htm)**
    Standard fields extend the InternalOrganizationUnit object for use in Automotive Cloud to represent account structures or departments within a company. This object is available in Automotive Cloud in API version 56.0 and later.
-   **[Automotive Cloud Fields on Lead](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_lead.htm)**
    Standard fields extend the Lead object for use in Automotive Cloud to represent information about the time frame during which a lead is interested in a transaction.
-   **[Automotive Cloud Fields on Product2](atlas.en-us.automotive_cloud.meta/automotive_cloud/sforce_api_objects_product2.htm)**
    Standard fields extend the Product2 object for use in Automotive Cloud to represent information about vehicle models, parts, and accessories.
