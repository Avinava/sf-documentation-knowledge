# Service Process Studio

Build enhanced customer service experiences through real-time integration with core banking functions and transactional data.

-   **[Service Process Studio Metadata Types](atlas.en-us.psc_api.meta/psc_api/srvprocstudio_metadata_apis_parent.htm)**
    Use the Service Process Studio metadata types to set up your Service Process use case. Define attributes, OmniScripts, flows, classes, and the Integration Definition.
-   **[Apex Reference](atlas.en-us.psc_api.meta/psc_api/srvprocstudio_apex_reference.htm)**
    This Apex reference goes into detail about the built-in Apex ConnectApi classes and interfaces.
-   **[Service Process Studio Invocable Actions](atlas.en-us.psc_api.meta/psc_api/service_process_studio_actions.htm)**
    Use actions to invoke Service Process Studio actions in a flow.
-   **[Service Process Studio Connect APIs](atlas.en-us.psc_api.meta/psc_api/service_process_studio_connect_apis.htm)**
    Use the Service Process Studio Connect API to create an extended Case object that includes custom data about a customer request such as a fee reversal or address update.
-   **[Service Process Studio Connect API Use Cases](atlas.en-us.psc_api.meta/psc_api/service_process_studio_connect_api_use_cases.htm)**
    Use the Service Process Studio connect API use cases to invoke connect API resources to create and manage instances of particular service processes without using OmniStudio.
