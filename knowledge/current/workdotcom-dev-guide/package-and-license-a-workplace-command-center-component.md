# Package and License a Workplace Command Center Component

To extend Workplace Command Center functionality, you package a solution that’s installed where Work.com and the Workplace Command Center package are installed. When deciding how to package your Workplace Command Center solution, consider the features that impact Work.com package and license dependencies.

-   **[Package Your Work.com Solution](atlas.en-us.workdotcom_dev_guide.meta/workdotcom_dev_guide/wdc_cc_packaging.htm)**
    Choose the most appropriate type of package based on your solution design and resulting dependencies.
-   **[Work.com Command Center Licensing](atlas.en-us.workdotcom_dev_guide.meta/workdotcom_dev_guide/wdc_cc_licensing.htm)**
    Design your solution with Salesforce platform and Work.com end-user license requirements in mind.
