# Prerequisites

# Prerequisites

Complete these prerequisites before you start developing with Metadata API.

-   To access Metadata API through the command line, [install Salesforce CLI](https://developer.salesforce.com/docs/atlas.en-us.260.0.sfdx_setup.meta/sfdx_setup/sfdx_setup_install_cli.htm).
-   To create a development environment, [sign up for Salesforce Developer Edition](https://developer.salesforce.com/signup). A Developer Edition org is a free development environment for building and testing solutions independent of production data.
-   Install [Salesforce Extensions for Visual Studio Code](https://marketplace.visualstudio.com/items?itemName=salesforce.salesforcedx-vscode). These tools provide features for working with development orgs (scratch orgs, sandboxes, and DE orgs), Apex, Aura components, and Visualforce.
-   Confirm that you have API Enabled permission and Modify Metadata Through Metadata API Functions permission or Modify All Data permission. If you don’t have these permissions set, [modify your metadata permissions](https://developer.salesforce.com/docs/atlas.en-us.260.0.api_meta.meta/api_meta/meta_modify_metadata_perm.htm).
-   [Enable Dev Hub in your org](https://developer.salesforce.com/docs/atlas.en-us.260.0.sfdx_dev.meta/sfdx_dev/sfdx_setup_enable_devhub.htm). Dev Hub allows you to create and manage scratch orgs so that you can develop without affecting your production data or metadata.
-   To allow access to protected resources such as production data and metadata, [authorize your org](https://developer.salesforce.com/docs/atlas.en-us.260.0.sfdx_dev.meta/sfdx_dev/sfdx_dev_auth.htm) .
