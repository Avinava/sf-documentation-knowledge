# Retail Execution

With Retail Execution, a sales manager can plan store visits for field representatives and analyze the business’s health across stores.

-   **[Standard Objects for Retail Execution](atlas.en-us.retail_api.meta/retail_api/sforce_api_objects_retail_overview.htm)**
    The consumer goods data model provides objects and fields to support retail execution.
-   **[Associated Objects for Retail Execution](atlas.en-us.retail_api.meta/retail_api/cg_retail_associated_objects.htm)**
    This section provides a list of objects associated to Retail Execution standard objects with their standard fields.
-   **[Custom Objects for Retail Execution](atlas.en-us.retail_api.meta/retail_api/sforce_api_objects_236_custom_objects.htm)**
    The consumer goods data model provides objects and fields to support Retail Execution (RE).
-   **[Retail Execution Standard Metadata APIs](atlas.en-us.retail_api.meta/retail_api/con_metadata_api.htm)**
    Metadata API enables you to access some entities and feature settings that you can customize in the user interface.
-   **[Retail Order Customization](atlas.en-us.retail_api.meta/retail_api/retail_order_ui_customization.htm)**
    Customize the order proposal list and the order screens, such as Order Header, Order Details, Order Items, Add Items, and Free Items by creating an Apex customization hook.
-   **[Sync Management](atlas.en-us.retail_api.meta/retail_api/cg_retail_sync_management.htm)**
    Sync Management is part of Retail Execution and it ensures that the information available in web portal and mobile application is the same.
-   **[Direct Store Delivery](atlas.en-us.retail_api.meta/retail_api/cg_retail_direct_store_delivery.htm)**
    Direct Store Delivery (DSD) is a supply chain model where a manufacturer sells goods directly to retailers. You can process orders, payments, deliveries, and returns by using DSD.
-   **[Einstein Visit and Activity Recommendation](atlas.en-us.retail_api.meta/retail_api/cg_retail_einstein_visit_and_activity_recommendation.htm)**
    This section provides a list of objects and fields that are related to Einstein Visit and Activity Recommendation.
