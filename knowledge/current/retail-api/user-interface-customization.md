# User Interface Customization

This section provides information on how to customize the TPM user interface.

-   **[UI Customization](atlas.en-us.retail_api.meta/retail_api/ui_customization.htm)**
    All managed packages provided UI components are replaceable by custom implementations if the default requirements are not suited for your implementation.
-   **[APEX Configuration](atlas.en-us.retail_api.meta/retail_api/apex_setup.htm)**
    In order to customize the Promotion Save process in APEX layer, you must introduce a callable APEX class as a customization hook.
-   **[LWC Configuration](atlas.en-us.retail_api.meta/retail_api/lwc_setup.htm)**
    In order to access the TPM promotion data in LWC layer, the tpm-promotion service component is used. Use the tpm-promotion component to hook into the promotion state and be able to extract data from the promotion (through events) and push new data to the promotion (through methods).
-   **[Apex Save Process Customization](atlas.en-us.retail_api.meta/retail_api/apex_save_customization.htm)**
    This section provides information on the different workflow process to customize the save process using APEX calls.
