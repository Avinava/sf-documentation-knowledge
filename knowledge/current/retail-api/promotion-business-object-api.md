# Promotion Business Object API

This section describes the base configuration provided for the Promotion Business Object.

The required records are automatically installed with the steps described within the section Installing Base Business Object API Configurations of [Business Objects API](atlas.en-us.retail_api.meta/retail_api/cg_retail_business_objects_api_wrapper.htm "Business Object API is a feature designed to facilitate integrations of external systems and processes with TPM related SObjects.").

-   **[Business Object API Entity](atlas.en-us.retail_api.meta/retail_api/cg_tpm_business_object_api_entity.htm)**
    The following entities are created as part of the default configuration to support Promotion use cases.
-   **[Business Object API Output Entities](atlas.en-us.retail_api.meta/retail_api/cg_tpm_business_object_api_output_entities.htm)**
    The following entities are provided for Promotion Business Objects API.
-   **[Business Object API Workflows](atlas.en-us.retail_api.meta/retail_api/cg_tpm_business_object_api_workflow.htm)**
    The following workflows are provided in the default configuration.
-   **[Workflows and Associated Workflow Steps](atlas.en-us.retail_api.meta/retail_api/cg_tpm_business_object_api_workflow_step.htm)**
    The following table provides information about the BO API Workflow and the associated Workflow Steps.
-   **[Business Object API Workflow Steps](atlas.en-us.retail_api.meta/retail_api/cg_tpm_business_object_api_workflow_steps.htm)**
    The following table provides information about the BO API Workflow Steps.
-   **[Business Object API Workflow Step Input Structures](atlas.en-us.retail_api.meta/retail_api/cg_tpm_business_object_api_workflow_step_input_structures.htm)**
    The following table provides technical details related to each workflow step.
-   **[Use Promotion Business Objects API](atlas.en-us.retail_api.meta/retail_api/cg_tpm_use_promotion_business_objects_api.htm)**
    Promotion Business Object API can be accessed through REST API, Aura Components, and Lightning Web Component.
-   **[Use Case: Add a New Workflow Step](atlas.en-us.retail_api.meta/retail_api/cg_tpm_use_case_add_a_new_workflow_step.htm)**
    In this use case, you will create a complete new workflow step that updates the Tactic Comment field with a new specified value that you will send as input.
