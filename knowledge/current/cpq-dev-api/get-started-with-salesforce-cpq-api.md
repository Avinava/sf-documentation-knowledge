# Get Started with Salesforce CPQ API

Want to start working with Salesforce CPQ API? Check out the CPQ data models, types of API, and quickstart guides.

| Available in: Salesforce CPQ Summer ’16 and later |
| --- |


![Note](/docs/resources/img/en-us/260.0?doc_id=images%2Ficon_note.png&folder=cpq_dev_api)

#### Note

The documentation in this developer guide is for the Salesforce CPQ managed package. If you're looking for developer documentation for Revenue Cloud, see [Get Started with Revenue Cloud Developer Resources](https://developer.salesforce.com/docs/atlas.en-us.260.0.revenue_lifecycle_management_dev_guide.meta/revenue_lifecycle_management_dev_guide/rlm_get_started.htm).

-   **[CPQ API Models](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_models.htm)**
    CPQ API references several CPQ data models. To use the Salesforce CPQ API, create classes for each data model in your org.
-   **[CPQ Quote API](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_pricing_parent.htm)**
    Manage quotes and quoting actions with CPQ quote API.
-   **[CPQ Configuration API](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_config_parent.htm)**
    Use the Salesforce CPQ Configuration API to configure and price product bundles.
-   **[CPQ Contract API](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_contract_parent.htm)**
    Use CPQ Contract API to amend and renew CPQ quotes.
-   **[Generate Quote Document API](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_generate_proposal.htm)**
    Creates and saves a CPQ quote document.
-   **[Service Router](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_service_router.htm)**
    SBQQ.ServiceRouter is a global Apex class serving as a single entry point for all API calls. You can use it for calls made by Apex code, Visualforce Remoting, or REST callouts.
-   **[CPQ API Quickstart Guide](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_quickstart.htm)**
    Review examples of integrating Salesforce CPQ API with your platform.
-   **[Disable CPQ Triggers in Apex](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_disable_apex_triggers.htm)**
    You can manually disable Salesforce CPQ and Salesforce Billing application logic when you update records. This process is helpful when you’re updating your own custom field. It’s also helpful when you update a record several times in one transaction and want triggers to run only on the last iteration.
-   **[Advanced Approvals API](atlas.en-us.cpq_dev_api.meta/cpq_dev_api/cpq_api_aa_intro.htm)**
    Customize your approval process with the API from the Advanced Approvals package.
