# Get Started with Scratch Org Snapshots

Install the required Salesforce DX tools, then enable Dev Hub and Scratch Org Snapshots in an org, usually your production org.

-   [Install Salesforce CLI](https://developer.salesforce.com/tools/salesforcecli).
-   [Enable Dev Hub](https://developer.salesforce.com/docs/atlas.en-us.260.0.sfdx_dev.meta/sfdx_dev/sfdx_setup_enable_devhub.htm) in your production org.
-   [Authorize your Dev Hub org](https://developer.salesforce.com/docs/atlas.en-us.260.0.sfdx_dev.meta/sfdx_dev/sfdx_dev_auth.htm). The Dev Hub org is the org you use to create and manage scratch orgs.
-   [Enable Scratch Org Snapshots in the Dev Hub org](atlas.en-us.sfdx_dev.meta/sfdx_dev/sfdx_dev_snapshots_enable_snapshots_devhub.htm "A snapshot must belong to the Dev Hub org that you’re using to create the scratch orgs.").
-   [Provide users with permissions to create snapshots](atlas.en-us.sfdx_dev.meta/sfdx_dev/sfdx_dev_snapshots_assign_permset_and_licenses.htm "Provide all non-admin Scratch Org Snapshots users with a supported license and access to the required scratch org and snapshot objects. Dev Hub (production org) admins can create and manage snapshots by default.").

-   **[Enable Scratch Org Snapshots in the Dev Hub Org](atlas.en-us.sfdx_dev.meta/sfdx_dev/sfdx_dev_snapshots_enable_snapshots_devhub.htm)**
    A snapshot must belong to the Dev Hub org that you’re using to create the scratch orgs.
-   **[Assign a License and Permissions to Snapshot Users](atlas.en-us.sfdx_dev.meta/sfdx_dev/sfdx_dev_snapshots_assign_permset_and_licenses.htm)**
    Provide all non-admin Scratch Org Snapshots users with a supported license and access to the required scratch org and snapshot objects. Dev Hub (production org) admins can create and manage snapshots by default.
