# Resources

Use these resources of Life Sciences Cloud Business APIs to enhance your Salesforce implementation or integrate with other applications.

-   **[Account Management](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/life_sciences_account_management_intro.htm)**
    Use these resources to manage customer accounts.
-   **[Account Manual Alignment](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_customer_manual_alignment.htm)**
    Manually align an account to a territory in the Life Sciences package to create a Provider Account Territory Info record.
-   **[Account User Territory Information (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_account_user_territory_information.htm)**
    Get user details, including territories and additional user information.
-   **[Advanced Provider Search](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_advanced_provider_search.htm)**
    Facilitate three distinct types of account searches: Healthcare Provider (HCP), Healthcare Organization (HCO), and license-based searches. For HCP and HCO searches, the API processes fields from the account, healthcare provider, and contact point address entities.The license-based searches use fields from the business license entity.
-   **[Book Slot Chain (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_book_slot_chain.htm)**
    Create service appointments for a care program enrollee in advanced therapy management based on the provided appointment slots.
-   **[Contact Encounter (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/hc_erm_resource_contact_encounter_business_api.htm)**
    Load details of contact encounters into the system.
-   **[Content Management](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/life_sciences_content_management_intro.htm)**
    Use these resources to manage customer content.
-   **[Care Program Enrollments (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/hc_careprogram_enrollment_intro.htm)**
    Use this API to enroll multiple patients in a care program.
-   **[Digital Verification (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_digital_verification.htm)**
    Perform verification for the digital verification records within advanced therapy management.
-   **[Download Provider (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_download_provider.htm)**
    Download Health Care provider data from an external system to a Salesforce org after a search is performed.
-   **[Benefit Verification](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/benefit_verification_api_intro.htm)**
    Use these resources to verify the health benefits covered by the patient's insurer. The call center agent can view the coverages in the patient's record page.
-   **[Intelligent Document Management](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/intelligent_doc_management_apis.htm)**
    Use the Intelligent Document Management resources to create or update records in Salesforce objects based on the content document records or received document records.
-   **[Patients (GET, POST, PUT)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_create_get_update_patients.htm)**
    Retrieve the FHIR patient resource record details. Create or update a patient record rather than making multiple separate calls to the objects in Salesforce.
-   **[Provider Search](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_ls_provider_search.htm)**
    Search for Healthcare providers and accounts based on a free-text input, either from internal Salesforce data or an external system. This API returns account information, including healthcare providers and matching contact point addresses.
-   **[Providers](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/hc_providers_api_intro.htm)**
    Use these resources to create practitioner or facility records, or update existing fields in provider records and insert new fields.
-   **[Sample Limits Validation (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_sample_limits_validation.htm)**
    Validate the samples requested during a visit against the sample limits for that account, product, and sample limit template.
-   **[Visits (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_create_visit.htm)**
    Create visits for sales representatives. For example, for a medical device visit, a sales representative visits the surgeon or a hospital location.
-   **[Work Type Lead Time (POST)](atlas.en-us.life_sciences_dev_guide.meta/life_sciences_dev_guide/connect_resources_work_type_lead.htm)**
    Get a list of work types and their associated lead times that are optimized across regions for a specific advanced therapy.
