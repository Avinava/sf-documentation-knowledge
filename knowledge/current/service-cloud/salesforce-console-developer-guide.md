# Salesforce Console Developer Guide

The Lightning Console JavaScript API and the Salesforce Console Integration Toolkit both interact with Salesforce console apps. This guide provides reference material for both.

Starting with API version 42.0 of the Salesforce Console Integration Toolkit, many of the methods used in existing Visualforce pages and third-party web tabs now work in Lightning Experience. Just point to the latest version of the toolkit script in your Visualforce pages or third-party web tabs. Third-party content must be allowlisted in the Trusted URLs list to be used in Lightning Experience. See [Classic Console API Methods Supported in the Lightning Console API](atlas.en-us.api_console.meta/api_console/sforce_api_console_lightning_classic_bridge_methods.htm "Visualforce pages and third-party web tabs that use some Salesforce Console Integration Toolkit methods work in Lightning Experience as-is. Just point to the latest version of the toolkit script in your Visualforce pages or third-party web tabs. Third-party content must be allowlisted with CSP directives via Trusted URLs. This table lists the Salesforce Console Integration Toolkit methods that you can use in Lightning Console JavaScript API starting with API version 42.0.") for a list of supported methods.

To use this guide, it helps if you have a basic familiarity with:

-   JavaScript
-   Visualforce
-   Web services
-   Software development
-   Salesforce console
-   Lightning
-   Lightning console apps

-   **[Get to Know Lightning Console](atlas.en-us.api_console.meta/api_console/sforce_api_console_js_lex_intro.htm)**
    Get started with the Salesforce console in Lightning Experience.
-   **[Get to Know the Utility Bar](atlas.en-us.api_console.meta/api_console/sforce_api_console_js_utility.htm)**
    The utility bar is a specialized type of Lightning page that gives your users quick access to common productivity tools. Utility bars are supported in Lightning Experience for desktop only.
