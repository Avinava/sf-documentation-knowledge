# Test Harness Prerequisites

To make full use of Test Harness, you need a complete working mobile development environment. If you haven’t set up your mobile development tools yet, see the following resources for guidance.

**Salesforce DX Setup**

-   [Set Up Your Development Environment](https://developer.salesforce.com/docs/component-library/documentation/en/lwc/lwc.install_setup_develop) for Lightning Web Components
-   [Set Up Your Development Environment](atlas.en-us.mobile_offline.meta/mobile_offline/dx_set_up_dev_environment.htm "Before you can create your first Lightning web component (LWC), or test a LWC in a mobile app, you must set up your development environment for mobile components.") for LWC Offline
-   [Preview Lightning Web Components on Mobile](https://developer.salesforce.com/docs/component-library/documentation/en/lwc/mobile_extensions)
-   [Set Up Xcode](https://developer.salesforce.com/docs/component-library/documentation/en/lwc/lwc.mobile_extensions_setup_ios_xcode) (required for iOS simulator)
-   [Set Up Android Studio](https://developer.salesforce.com/docs/component-library/documentation/en/lwc/lwc.mobile_extensions_setup_android_studio) (required for Android emulator)

![Note](/docs/resources/img/en-us/260.0?doc_id=images%2Ficon_note.png&folder=mobile_offline)

#### Note

You don’t need to set up both Xcode and Android Studio unless you want to use Test Harness in virtual devices on both platforms.

**Third-Party Developer Tools**

-   [Xcode Overview](https://developer.apple.com/documentation/xcode) (Apple developer documentation)
-   [Meet Android Studio](https://developer.android.com/studio/intro) (Google developer documentation)
