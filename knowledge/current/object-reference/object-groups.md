# Object Groups

Objects can be grouped in any number of ways. Object groups are based on where data is stored, how it's used, and which licenses are required to access it.

-   Salesforce Common Objects—available for any org
-   Salesforce Cloud Objects—available for specific clouds or features
-   High-Scale Objects—available when purchased for specific clouds
-   External Data—stored outside Salesforce

-   **[Salesforce Common Objects](atlas.en-us.object_reference.meta/object_reference/sforce_api_concepts_common_objects.htm)**
    Common objects are available in any standard org for any cloud. They don’t require any special licensing.
-   **[Salesforce Cloud Objects](atlas.en-us.object_reference.meta/object_reference/sforce_api_concepts_cloud_objects.htm)**
    Cloud objects are used only by a specific cloud. These objects sometimes have different limitations, APIs, or querying behavior than standard common objects.
-   **[Salesforce High-Scale Objects](atlas.en-us.object_reference.meta/object_reference/sforce_api_concepts_high_scale_objects.htm)**
    Use high-scale objects when you have a large number of transactions, large objects, or you must store data over a long period. These objects sometimes have different limitations, APIs, or querying behavior than Salesforce common objects do.
-   **[Salesforce External Data Objects](atlas.en-us.object_reference.meta/object_reference/sforce_api_concepts_external_data_objects.htm)**
    You can store data outside Salesforce and still access in your org or Data Cloud.
