Contains the error codes and error code messages that are applicable to all Insurance APIs across all Insurance families.

| Error Code | HTTP Response Code | Message |
| --- | --- | --- |
| SF-0001-0001 | 400 | Specify a context ID of type {expectedContextType}. |
| SF-0001-0002 | 400 | The {dateField1} must be after the {dateField2}. |
| SF-0001-0003 | 400 | Specify a date in yyyy-MM-dd format. |
| SF-0001-0004 | 400 | Specify an input for {inputField}. |
| SF-0001-0005 | 400 | Specify a valid input for {inputField}. |
| SF-0001-0006 | 400 | Specify a valid context ID. |
| SF-0001-0007 | 400 | Your input is invalid. Specify a valid payload and try again. |
| SF-0001-0008 | 400 | Specify the default picklist value for an attribute in {entity} and try again. |
| SF-0001-0009 | 400 | Something went wrong. Check the following error and try again: {errorMessage}. If the issue persists, contact your Salesforce admin for help. |
| SF-0001-0010 | 400 | An internal error has occurred. Contact your Salesforce admin for help. |
| SF-0001-0011 | 400 | Looks like you don't have access to {operation} this {entity}. Your Salesforce admin can help with that. |
| SF-0001-0012 | 400 | Field Excluded From Update. Error {errorMessage} |
| SF-0001-0013 | 400 | {errorMessage}. Check field names are correct and user has access to the specified fields |
